CSS FAQ

符合Web标准的网站首先一点条件是网页能够通过W3C的代码校验，W3C提供了一个帮助使用使用者校验自己网站各方面语法程序，W3C目前提供了HTML / XHTML / CSS / RDF / P3P / XML /等多种标记语言的校验工具，如果你使用了这些语言来构建你的网站及应用，可以使用相关的工具进行语法校验。通过校验是学习Web标准的第一步，如果你在设计符合Web标准的网站中最后能够通过W3C校验，那么证明你的网页在Web标准的语法层面的使用上是没有问题的。但是仅仅为通过W3C校验而设计的网页是没有价值的，符合Web标准的另一层含义是使用Web标准中的各项技术将网站表现与内容完全分离，从根本上改变现有网站的结构为网站带来革新性的变化。所以我们认为通过W3C校验是学习与测试自己对Web标准中的技术性语法掌握的基础，而真正需要符合Web标准，还是需要不断学习与提高在网站架构设计上的经验，实现网站表现与内容的分离。

答案是肯定的，HTML也是W3C的标准之一，之所以推荐使用XHTML是因为HTML的设计形式已经不能满足表现与内容分离式的网站架构，但是如果继续使用HTML来构建的话，从最终目标上来说是没有差别，通过完整的HTML标记语言，也同样能够通过W3C的校验，最终网页的显示也不会有什么问题。不过在这里还是继续向大家推荐使用XHTML，必尽XHTML提供更合理的结构，在Web标准设计之中会得到方便，在未来也方便顺利过渡到XML。

XML是未来数据的描述格式，目前在应用软件及Web应用的开发中已经大量使用XML技术进行数据交换与整合，如果技术成熟，XML会取代XHTML/HTML成为未来的网站构建中实现内容层的一环。但就目前而言，XML在实现网站表现层上有一定困难，使用XML+CSS或XSTL技术构建网站本身技术上难度较高，而XML自由的标记结构也对网站设计者提高了门槛，

浏览器对XML的直接支持也不太完善，所以XML目前的应用主要在Web应用及软件中用于数据交换，总的来说目前不适于直接应用XML来构建网站。不过总是有一些先例。目前网络上有很多Blog系统的rss功能就是基于XML的一种应用方式，目前有许多网站，包含微软的MSN Space，都已经为Blog中的rss功能提供单独的样式表以方便当用户直接用浏览器打开Rss文件时提供具有样式支持的rss浏览方式，也就是XML网页设计的一个初步实例，相信在未来会有更方便更简便的方法帮助我们转向XML。

本书正是一门帮助大家用CSS进行网站设计的入门指导读物，事实上CSS并没有想象中那么复杂，CSS只是相对于HTML而言提供了更多的控制标签，而这些标签的使用基本上简单易学的，使用CSS进行网站设计在修改上比起纯HTML设计要更为简单快捷，只需要修改CSS文件就可以控制整个布局及样式的变化。因此我们认为CSS是一门简单有效，易读易学的语言，对于拥有网站设计基础的人来说过渡将会非常方便，CSS将为网站设计者提供更自由的创作空间。
